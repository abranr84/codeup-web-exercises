What is a FONT STACK and how should the best one be chosen?

A font stack is a list of fonts and font keywords used to let a browser know our preferred and fallback fonts. A font stack is a list of fonts in the CSS font-family declaration. The fonts are listed in order of preference that you would like them to appear in case of a problem, such as a font not loading. Font names are separated by commas and multiple-word font names are enclosed in quotation marks. You can add as many fonts as you'd like, as long as they are separated by commas. The browser will try to load the first font specified. If that fails, it will run down the line trying each font until it finds one that it can use. A font stack usually ends with a generic font classification (serif or sans-serif). There’s a basic formula to creating a good font stack: ‘Preferred Font’, ‘Next best thing’, ‘Something common and sorta close’, ‘Similar Web-safe’, ‘Generic font’. There’s nothing wrong with having more than one font for any of those, but try to keep your font stack reasonably short (six to ten fonts is a pretty good maximum number).

What is the difference between SERIF and SANS SERIF font?

The decorative strokes: A serif is a decorative stroke that extends off the end of a letterform.

Serif fonts are sometimes considered more classic or formal.

Sans-serif fonts are often considered more minimalist or casual.

You will often find that print publications like books and newspapers will use serif fonts, while digital publications or magazines favor sans-serif fonts. Some people believe that a serif font choice is better for reading text at small sizes in printed copy (like books or newspapers), while the sans-serif style is easier to read in digital mediums.
